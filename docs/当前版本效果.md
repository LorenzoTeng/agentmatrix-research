## 二、能做什么

### 2.1 因子注册与管理

```
新因子 idea → 填 Spec（18 字段）→ 自动进入管线
```

215 个因子已完成标准化注册，按 10 大类组织。新增因子只需填充 FactorResearchSpec 模板即可被 CLI、CI、验证管线统一消费。

### 2.2 跨引擎真值验证 + 自动诊断

```
GM 计算 → 与 JQ 真值对比 → NC 分类过滤 → 诊断引擎定位根因
```

- **NC 分类器**：3 层过滤（因子级 NC → 额外 NC → 股票级 NC），129/215 因子已分类
- **诊断引擎**：12 种检测器自动识别根因——TTM 转换、单位缩放、日期语义、维度错误、early_filer...
- **v78 验证**：31 因子 × 沪深 300，NC 过滤后 MISMATCH = 0

### 2.3 AI 因子挖掘 + 自动验证

```
自然语言 idea → AI 生成表达式 → bridge 映射 → 自动验证 → 反馈 AI 迭代
```

- **13 种表达式模式**：MOMENTUM、VOLUME_RATIO、VOLATILITY、MOVING_AVERAGE、PRICE_RATIO、CORRELATION、DELTA、复合表达式...
- **多 LLM 支持**：OpenAI、DeepSeek、Qwen、Zhipu、Moonshot、自定义
- **双库路由**：OHLCV 表达式 → ai_factors（Qlib IC），GM 字段表达式 → jq_gm（VM IC）
- **反馈闭环实验**：Round 1 70% PASS → Round 2 **100% PASS**

### 2.4 CI 质量门禁

每次代码变更自动触发：
- **覆盖率检查**：Spec 必须有 gm_field、display_name、测试用例
- **回归检测**：因子值变化 > 5% 阻断合并（Path A stub 25 表达式 + Path B GM 308 因子）
- **管线验证**：proof-batch 端到端跑通

### 2.5 数据源无关计算

`_compute_directly()` 只依赖列名（close / volume / high / low），不绑定 GM SDK。Wind、Tushare、Bloomberg 等面板只要列名对齐即可直接接入。

### 2.6 诊断知识库（58 版本经验沉淀）

| 检测器 | 能发现什么 | 关键教训 |
|--------|-----------|---------|
| TTM 转换 | GM 值稳定低于 JQ ~28% | 用中位数检测，不用均值（离群值污染方差） |
| 单位缩放 | 增长率 *100 vs 原生百分比 | GM 已返回百分比，重复 *100 导致 100x 偏差 |
| 日期语义 | date 返回 FY 年报而非 Q3 | early_filer 需要 statDate 强制指定季度 |
| 级联失败 | 一个 API 字段无效拖垮后续 | 先查 NC 再查 BOTH_NA（v66 教训） |
| 波动率年化 | Std 未乘 sqrt(252) | daily std → annual 需要 *sqrt(252) |
| 股票级 NC | 少数股票数据源差异 | <30 条且差异 <30% → 标记而非整体标 NC |

## 四、三条使用路径

### 路径 A：手工因子验证

```
1. 在 specs.py 注册新因子
2. 运行 run-jq-gm-demo（本地）
3. 导出 JQ 真值 → 运行 proof-batch（VM）
4. NC 分类器 + 诊断引擎自动分析
5. 看 HTML/MD 报告
```

### 路径 B：AI 批量挖掘

```
1. auto-mine --theme "低波动质量股" --count 10
2. verify_candidates() 批量验证
3. feedback_to_prompt() 提取结构化反馈
4. auto-mine --feedback "<上轮反馈>" --count 10
5. 通过因子 → expression_to_spec() → 入库
```

> ✅ GM SDK 端到端验证通过（2026-06-24）：Price 路径 4/4 OK + Fundamental 路径 (pe_ttm/roe_ttm) 真实计算成功。脚本：`scripts/bridge_e2e_verify.py`

### 路径 C：CI 自动化

```
1. 提交 PR → CI 自动触发
```

## 五、局限性

| 局限 | 说明 | 解决方向 |
|------|------|---------|
| stub 模式 | 无 GM SDK 时 compute 返回 NaN | VM 端运行真实验证 |
| Qlib 依赖 | auto-mine 的 IC 评估需要 Qlib 数据 | 安装 Qlib + 下载 cn_data |
| 复合表达式 | parser 识别了类型但只计算核心分量 | 扩展 _compute_directly |
| 无真值因子 | 188/215 暂无外部真值数据 | JQ SDK 拉取 + NC 分类 |
| 未合入上游 | 当前在 fork，等待 review | PR #23 已提交 |
